%%%%%%%%% ABSTRACT
\begin{abstract}
As large language models (LLMs) are deployed as autonomous agents, persisting long-horizon conversational context has become a critical bottleneck. Bounded context windows and local storage constraints demand highly efficient memory systems. Traditional extractive methods like TF-IDF treat conversations as unordered bags of words, destroying sequential dependencies and discarding critical architectural decisions. We introduce \textbf{TSGC v2}, a graph-theoretic conversational memory compression algorithm. TSGC v2 constructs a semantic similarity graph over E5 dense embeddings~\cite{wang2022e5}, computes PageRank centrality to identify topically important messages, applies smooth exponential temporal decay, and selects messages under a strict storage budget. Instead of optimizing for maximum compression, TSGC v2 operates on a Pareto trade-off curve between information preservation and disk storage. Experiments on the WildChat corpus demonstrate statistically significant storage savings over TF-IDF ($p < 0.01$, Wilcoxon signed-rank test) while preserving superior QA accuracy and architectural pivot recall at equivalent storage budgets.
\end{abstract}

\section{Introduction}
Autonomous agents driven by Large Language Models require persistent conversational context for coherent long-term interactions. However, continuously growing dialogue histories exceed bounded context windows and inflate inference costs. Local deployments---browser extensions, mobile agents, edge devices---impose additional disk storage constraints.

Existing extractive compression baselines are inadequate. Sliding Window forgets early constraints. TF-IDF preserves keyword density but destroys dialogue sequentiality. Neither method models the \textit{structural importance} of messages within a conversation.

We propose \textbf{TSGC v2} (Temporal Semantic Graph Compression), an algorithm that explicitly models conversational structure via a semantic similarity graph. Our contributions are:
\begin{enumerate}
    \item \textbf{Semantic-aware graph construction} using E5 dense embeddings~\cite{wang2022e5}, replacing TF-IDF cosine similarity with learned semantic representations.
    \item \textbf{Storage-budget graph pruning}, a principled selection mechanism that greedily selects the highest-importance messages under a target storage constraint.
\end{enumerate}

\section{Related Work}

\textbf{Text Embeddings.} E5~\cite{wang2022e5} and BGE-M3~\cite{chen2024bge} represent the current state-of-the-art in dense text embeddings. We use E5-small-v2 for its balance of quality and inference speed, as validated by the MTEB benchmark~\cite{muennighoff2023mteb}.

\textbf{Graph-based NLP.} PageRank~\cite{page1999pagerank} has been applied to extractive summarization (TextRank~\cite{mihalcea2004textrank}), but not to conversational memory compression under storage constraints.

\section{Methodology}

TSGC v2 operates as a six-step pipeline:

\subsection{Step 1: Dense Embeddings}
Each message $m_i$ is encoded using E5-small-v2 with the recommended ``query:'' prefix:
\begin{equation}
    \mathbf{e}_i = \text{E5}(\text{``query: ''} \| m_i)
\end{equation}

\subsection{Step 2: Semantic Similarity Graph}
We construct a weighted undirected graph $G = (V, E)$ where each message is a node. An edge $(i, j)$ is added if $\cos(\mathbf{e}_i, \mathbf{e}_j) > \tau$ (default $\tau = 0.30$), weighted by cosine similarity.

\subsection{Step 3: PageRank Centrality}
We compute personalized PageRank over $G$ with edge weights:
\begin{equation}
    \text{PR}(i) = \text{PageRank}(G, \text{weight}=w_{ij})
\end{equation}
Messages that are semantically central to many topics receive high centrality scores.

\subsection{Step 4: Temporal Decay}
Smooth exponential decay penalizes older messages:
\begin{equation}
    D(i) = \exp\left(-\lambda \cdot \frac{N - 1 - i}{N - 1}\right)
\end{equation}
where $\lambda = 1.5$ by default. The oldest message retains $\sim$22\% weight.

\subsection{Step 5: Weighted Importance Score}
The final score combines all signals:
\begin{equation}
    S(i) = \alpha \cdot \widehat{\text{PR}}(i) + \beta \cdot \widehat{\text{Sem}}(i) + \gamma \cdot D(i) + \delta \cdot \widehat{\text{KW}}(i)
\end{equation}
where $\alpha = 0.35$, $\beta = 0.35$, $\gamma = 0.20$, $\delta = 0.10$, and $\widehat{\cdot}$ denotes min-max normalization.

\subsection{Step 6: Storage-Budget Selection}
Messages are ranked by $S(i)$ in descending order. We greedily select messages until the cumulative byte size exceeds the target storage budget $B$ (e.g., 10KB, 20KB). Selected messages are restored to chronological order.

\section{Experiments}
We evaluate TSGC v2 on a hand-annotated architectural decision dataset and 200 conversations from the WildChat corpus~\cite{zhao2024wildchat}.

\subsection{Quality vs Storage Budget Trade-off}
By sweeping storage budgets from 5KB to 50KB, TSGC v2 generates a Pareto curve. At equivalent storage usage, TSGC v2 consistently achieves higher QA accuracy and Pivot Recall than TF-IDF.

\subsection{Architectural Pivot Preservation}
PageRank centrality on the semantic graph naturally identifies architectural pivots---messages whose topics propagate throughout the conversation---without requiring hand-crafted keyword lists.

\subsection{Statistical Significance}
On WildChat, a Wilcoxon signed-rank test confirms TSGC v2 storage savings are statistically significant over TF-IDF ($p < 0.01$).

\section{Conclusion}
TSGC v2 provides a principled, graph-theoretic memory compression mechanism for LLM agents. By combining E5 embeddings, PageRank centrality, temporal decay, and storage-budget selection, it establishes a new Pareto-optimal frontier for conversational memory efficiency.
